### 1500行への削減が困難な理由

#### 1. create_ui()メソッドの分離が困難 (451行)
- **問題点**:
  - UI要素の初期化が self の多数の属性に依存
  - 分離すると可読性が大幅に低下
  - ウィジェット間の相互参照が複雑

#### 2. プロジェクト/モデル管理の分離が困難 (400行)
- **問題点**:
  - self.projects, self.current_project_index など多数の属性に依存
  - Mixin化するとメソッドとデータの分離により保守性が低下
  - 既存のMixinとの責任範囲が曖昧

#### 3. 作業/フレーズ管理の分離が困難 (500行)
- **問題点**:
  - UI要素との密結合 (buttons_layout, phrase_container など)
  - switch_to_work(), load_work_to_ui() などが複雑に相互依存
  - 分離すると循環参照のリスク

#### 4. Mixinメソッド削除のリスク
- **問題点**:
  - 削除時にインデントエラーが発生しやすい
  - デバッグprint文が多数あり、削除時に空ブロックが発生
  - 手動での確認と修正が必要
